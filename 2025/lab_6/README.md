# Лабораторная работа №6  
## Сбор данных через API (JSON)

### Цель работы
Изучить принципы получения данных с веб-сайта с использованием API,
обработку JSON-ответов, работу с асинхронными запросами и сохранение
результатов в файл.

---

### Описание задания
В лабораторной работе реализован сбор данных о товарах с сайта
**wildberries.ru** с использованием API поиска товаров.
Данные загружаются постранично, так как на одной странице API
возвращает ограниченное количество товаров.

---

### Возможное возникновение ошибки 429
Наличие ошибки 429 не является ошибкой логики программы.
Ошибка 429 возникает из-за ограничений API и антибот-защиты Wildberries.
Это ограничение, установленное сервером сайта,
и оно не зависит от корректности клиентского кода.

В программе реализованы меры для уменьшения вероятности ошибки 429:
- задержки между последовательными запросами;
- повторные попытки выполнения запроса;
- увеличение времени ожидания между повторными запросами;
- обработка исключительных ситуаций без аварийного завершения программы.

---

### Логика работы программы
1. Выполняется запрос к API Wildberries с сортировкой товаров
   по убыванию цены.
2. Определяется общее количество товаров по запросу.
3. Загружаются данные с первых трёх страниц.
4. Данные со всех страниц объединяются в один массив.
5. Объекты преобразуются и оставляются только необходимые поля:
   - бренд
   - название товара
   - количество оценок
   - рейтинг поставщика
   - ссылка на товар
   - текущая и базовая цена
   - характеристики товара
6. Результат сохраняется в файл `products.json`.

---

### Используемые технологии
- **Node.js**
- **axios** — выполнение HTTP-запросов к API
- **fs/promises** — запись данных в файл
- **async / await** — асинхронная обработка запросов
